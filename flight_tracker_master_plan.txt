# Flight Price Tracker - Master Plan & Agent Prompt

## 📌 Project Overview
We are building a modern, ultra-minimalist Flight Price Tracker. The application consists of a high-performance **Python (FastAPI)** backend handling scheduling, data fetching, database storage, and notifications, paired with a clean **HTML/JS (Tailwind CSS)** frontend. 
The frontend must be architected as a Single Page Application (SPA) compatible with **CapacitorJS** so it can be natively compiled into an **Android APK**.

---

## 🛠 Tech Stack Specification
- **Backend:** Python 3.11+, FastAPI, Uvicorn.
- **Database:** SQLite with SQLModel / SQLAlchemy ORM.
- **Scheduler:** `apscheduler` to fetch prices 3 times a day automatically.
- **Flight Data API:** Skyscanner (via RapidAPI) or a Scraping/Official Google Flights API (e.g., SerpApi / FlightAPI). Provide a robust *Mock Service* for local testing first.
- **Notifications:** - **Email:** Ingesting SMTP or Resend API.
  - **Push Notifications:** Firebase Cloud Messaging (FCM) or Web Push API (compatible with mobile webviews).
- **Frontend:** Single-Page HTML5, Vanilla JS (or HTMX/Alpine.js for simplicity), Tailwind CSS via CDN or local build.
- **Mobile Wrapper:** CapacitorJS (to target Android APK output).

---

## 🗂 Architecture Design & Flow
1. **Scraper Service:** Runs periodically -> calls Flight API -> parses response.
2. **Analysis Engine:** Compares current price against `lowest_historical_price` or `target_price` in SQLite.
3. **Trigger:** If price drops, fires asynchronous tasks to send Email + Push Notification via FCM.
4. **API Layer:** FastAPI exposes endpoints for the mobile frontend to add/remove tracked routes, view history, and register FCM tokens.
